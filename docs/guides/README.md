```
1. QUICK_START_VIBE_CODING.md 읽기

3. 막히면 PROMPT_CHEATSHEET.md 참조
```

### 시나리오 2: "신규 컴포넌트를 만들어야 합니다"

2. Step 1~5 따라하기

```

### 시나리오 3: "기존 프롬프트를 수정해야 합니다"
```

2. VIBE_CODING_WORKFLOW.md의 "기존 프롬프트 검증 및 수정" 참조
3. 트러블슈팅 섹션 확인
```

### 시나리오 4: "뭔가 잘못됐는데 어떻게 고쳐야 할지 모르겠습니다"
```
1. VIBE_CODING_WORKFLOW.md의 "트러블슈팅" 섹션
2. QUICK_START_VIBE_CODING.md의 "트러블슈팅" 섹션
3. ../rules/PROMPT_WIREFRAME_UI_SEPARATION.md의 예시 확인
```

### 시나리오 5: "팀원에게 프로세스를 설명해야 합니다"
```
1. README.md (프로젝트 루트) 보여주기
2. QUICK_START_VIBE_CODING.md 공유
3. VIBE_CODING_WORKFLOW.md의 "팀 협업" 섹션 참조

- `PROMPT_CHEATSHEET.md` - 빠른 참조용

### Tip 2: 명령어 복사
모든 가이드에는 복사 붙여넣기용 명령어가 포함되어 있습니다.
```
"이렇게 따옴표로 감싸진 명령어"를 복사해서 AI에게 요청하세요.
```

```
첫날: QUICK_START_VIBE_CODING.md
둘째날: PROMPT_CHEATSHEET.md
셋째날: VIBE_CODING_WORKFLOW.md
```

`PROMPT_CHEATSHEET.md`를 프린트해서 책상에 두면 편리합니다.

## ❓ FAQ

**Q: 어떤 문서부터 봐야 하나요?**
A: `QUICK_START_VIBE_CODING.md`부터 시작하세요.

**Q: 명령어를 외워야 하나요?**
A: 아니요, 복사 붙여넣기하세요. `QUICK_START_VIBE_CODING.md`에 모두 있습니다.

**Q: Wireframe과 UI가 헷갈려요.**
A: `PROMPT_CHEATSHEET.md`의 체크리스트를 참조하세요.
   또는 `../rules/PROMPT_WIREFRAME_UI_SEPARATION.md`의 예시를 보세요.

**Q: AI가 제대로 동작하지 않아요.**
A: `VIBE_CODING_WORKFLOW.md`의 "트러블슈팅" 섹션을 확인하세요.
