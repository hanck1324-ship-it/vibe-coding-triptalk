            - @01-common.mdc

==============================================

조건-파일경로) 참고할 API 문서경로: https://main-practice.codebootcamp.co.kr/graphql
조건-파일경로) 구현될 페이지 경로: src/app/boards/main/[boardId]/edit/page.tsx
조건-파일경로) 구현될 HOOK 파일경로: src/components/boards-write/hook.ts (수정 기능 추가)
조건-파일경로) 구현될 GQL 파일경로: src/components/boards-write/queries.ts (updateBoard 뮤테이션)
조건-파일경로) (참고) 공통 GQL 타입: src/commons/graphql/graphql.ts (codegen으로 생성된 타입 사용)

==============================================

핵심요구사항) 게시글 수정 기능을 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.

        1) 라이브러리조건
            - API: @apollo/client의 useQuery, useMutation 훅을 사용할 것.
            - 타입: @/commons/graphql/graphql에서 codegen으로 생성된 타입을 사용할 것.
            - 라우팅: next/navigation의 useRouter, useParams 훅을 사용할 것.
            - 상태 관리: React의 useState 훅을 사용할 것.

        2) 기능 요구사항
            - 1. (GQL) src/components/boards-write/queries.ts 파일에 이미 정의된 updateBoard 뮤테이션을 확인하고 사용할 것.
                - Mutation: updateBoard
                - Variables: boardId (ID!), password (String!), updateBoardInput
                - 응답데이터: _id, writer, title, contents 등

            - 2. 수정 페이지 초기 데이터 로딩
                - useParams를 사용하여 boardId 가져오기
                - useQuery를 사용하여 fetchBoard 쿼리로 기존 게시글 데이터 조회
                - 조회한 데이터를 폼에 초기값으로 설정
                - loading 상태 처리

            - 3. (HOOK) 게시글 수정 기능 구현
                - "use client" 지시어 추가
                - 게시글 수정을 위한 상태 관리
                - updateBoard 뮤테이션 실행
                - 패스워드 검증
                - 성공 시 상세 페이지로 이동
                - 실패 시 에러 메시지 표시

            - 4. UI 업데이트
                - 기존 작성 폼 재사용
                - 초기값 바인딩
                - 수정 버튼 텍스트 변경 ("등록하기" → "수정하기")

        3) 에러 처리 요구사항
            - 로딩 중 상태 표시
            - 데이터가 없을 경우 처리
            - 패스워드 불일치 시 에러 처리
            - 뮤테이션 실패 시 에러 메시지 표시

        4) 최적화 요구사항
            - 불필요한 리렌더링 방지
            - 성공 후 적절한 페이지 이동
