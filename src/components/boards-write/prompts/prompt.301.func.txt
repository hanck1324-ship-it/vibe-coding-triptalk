핵심요구사항) 게시글 작성 및 수정 기능을 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.

        1) 라이브러리조건
            - API: @apollo/client의 useMutation, useQuery 훅을 사용할 것.
            - 타입: @/commons/graphql/graphql에서 codegen으로 생성된 타입을 사용할 것.
            - 라우팅: next/navigation의 useRouter 훅을 사용할 것.
            - 상태 관리: React의 useState 훅을 사용할 것.
            - 모달: antd의 Modal 컴포넌트 사용 (알림용)

        2) 기능 요구사항

            - 1. (HOOK) hook.ts 파일에 useBoardWrite 커스텀 훅 생성
                - "use client" 지시어 추가
                - Props로 isEdit (boolean) 받기 (작성/수정 모드 구분)
                
                - 상태 관리:
                    - writer, password, title, contents (필수 입력값)
                    - writerError, passwordError, titleError, contentsError (에러 메시지)
                    - zipcode, address, addressDetail (주소 정보)
                    - youtubeUrl (유튜브 링크)
                    - isActive (등록 버튼 활성화 여부)
                    - isPostcodeModalOpen (우편번호 검색 모달)
                    - isAlertModalOpen, modalContents (알림 모달)
                
                - 입력 핸들러 함수:
                    - onChangeWriter: 작성자 입력 시 유효성 검증
                    - onChangePassword: 비밀번호 입력 시 유효성 검증
                    - onChangeTitle: 제목 입력 시 유효성 검증
                    - onChangeContents: 내용 입력 시 유효성 검증
                    - onChangeAddressDetail: 상세주소 입력
                    - onChangeYoutubeUrl: 유튜브 URL 입력
                
                - 우편번호 검색:
                    - handleTogglePostcodeModal: 모달 열기/닫기
                    - handleCompletePostcode: 주소 선택 완료 시 zipcode, address 상태 업데이트
                
                - 유효성 검증:
                    - 작성자: 공백 불가 (에러 메시지: "작성자를 입력해주세요.")
                    - 비밀번호: 공백 불가 (에러 메시지: "비밀번호를 입력해주세요.")
                    - 제목: 공백 불가 (에러 메시지: "제목을 입력해주세요.")
                    - 내용: 공백 불가 (에러 메시지: "내용을 입력해주세요.")
                
                - isActive 조건:
                    - writer && password && title && contents가 모두 입력되었을 때 true
                    - useEffect로 의존성 배열에 위 4개 값을 넣어 자동 업데이트

            - 2. (GQL) 게시글 작성 뮤테이션
                - queries.ts의 CREATE_BOARD 뮤테이션 사용
                - useMutation으로 createBoard 실행
                - Variables:
                    - createBoardInput: {
                        writer: string
                        password: string
                        title: string
                        contents: string
                        youtubeUrl?: string (선택)
                        boardAddress?: {
                          zipcode: string
                          address: string
                          addressDetail: string
                        } (선택)
                      }
                
                - onClickSubmit 함수:
                    - 필수 입력값 유효성 재검증
                    - createBoard 뮤테이션 실행
                    - 성공 시:
                        - 알림 모달 표시: "게시글이 등록되었습니다."
                        - 확인 버튼 클릭 시 게시글 목록 페이지로 이동 (/boards)
                    - 실패 시:
                        - 에러 메시지 모달 표시

            - 3. (GQL) 게시글 수정 기능 (isEdit === true일 때)
                - useParams로 boardId 가져오기
                - useQuery로 FETCH_BOARD 쿼리 실행하여 기존 데이터 로드
                - 로드한 데이터로 초기값 설정 (defaultValue)
                
                - UPDATE_BOARD 뮤테이션 사용
                - onClickUpdate 함수:
                    - 필수 입력값 유효성 재검증
                    - updateBoard 뮤테이션 실행
                    - Variables: boardId, password, updateBoardInput
                    - 성공 시:
                        - 알림 모달: "게시글이 수정되었습니다."
                        - 해당 게시글 상세 페이지로 이동 (/boards/[boardId])
                    - 실패 시:
                        - 패스워드 불일치 등 에러 메시지 표시

        3) 알림 모달 처리
            - isAlertModalOpen: 모달 표시 여부
            - modalContents: 모달에 표시할 메시지
            - handleOk: 확인 버튼 클릭 시 처리
                - 모달 닫기
                - 필요 시 페이지 이동

        4) 에러 처리 요구사항
            - 입력값 에러는 각 input 하단에 빨간색 텍스트로 표시
            - API 에러는 알림 모달로 표시
            - try-catch로 예외 처리

        5) 반환값 (return)
            - writer, writerError, onChangeWriter
            - password, passwordError, onChangePassword
            - title, titleError, onChangeTitle
            - contents, contentsError, onChangeContents
            - onClickSubmit, onClickUpdate
            - isActive, data (수정 모드에서 사용)
            - zipcode, address, addressDetail, youtubeUrl
            - onChangeAddressDetail, onChangeYoutubeUrl
            - isPostcodeModalOpen, handleTogglePostcodeModal, handleCompletePostcode
            - isAlertModalOpen, modalContents, handleOk

        6) 테스트 체크리스트
            - [ ] 필수 입력값이 모두 입력되면 등록 버튼이 활성화되는가?
            - [ ] 유효성 검증이 정상적으로 작동하는가?
            - [ ] 유튜브 URL이 입력되고 저장되는가?
            - [ ] 주소 검색이 정상적으로 작동하는가?
            - [ ] 게시글 작성이 성공적으로 완료되는가?
            - [ ] 게시글 수정이 정상적으로 작동하는가?
