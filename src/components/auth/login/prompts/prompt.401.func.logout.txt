아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @01-common.mdc
            - @04-func.mdc

==============================================

조건-파일경로) 구현될 HOOK 파일경로: src/components/auth/login/hook.ts (로그아웃 기능 추가)
조건-파일경로) 또는 공통 HOOK: src/commons/auth/useAuth.ts (인증 관련 훅)

==============================================

핵심요구사항) 로그아웃 기능을 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.

        1) 라이브러리조건
            - 상태 관리: React의 useState, useEffect 훅을 사용할 것.
            - 라우팅: next/navigation의 useRouter 훅을 사용할 것.
            - 저장소: localStorage 또는 sessionStorage 사용

        2) 기능 요구사항
            - 1. 로그아웃 기능 구현
                - "use client" 지시어 추가
                - 저장된 accessToken 제거
                - 로그인 상태 초기화
                - 로그인 페이지로 리다이렉트

            - 2. 인증 상태 관리
                - 로그인/로그아웃 상태를 전역적으로 관리
                - Context API 또는 전역 상태 관리 라이브러리 사용 고려

            - 3. UI 업데이트
                - 로그아웃 버튼 추가 (Navigation 또는 Header)
                - 로그인 상태에 따라 버튼 표시/숨김
                - 로그아웃 확인 다이얼로그 (선택사항)

        3) 보안 요구사항
            - 토큰 완전 제거 (localStorage, sessionStorage, 메모리)
            - 세션 쿠키 제거 (사용하는 경우)
            - 캐시 정리 (선택사항)

        4) 최적화 요구사항
            - 불필요한 리렌더링 방지
            - 로그아웃 후 적절한 페이지 이동
