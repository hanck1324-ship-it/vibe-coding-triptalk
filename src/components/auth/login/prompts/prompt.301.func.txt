        1) 라이브러리조건
            - API: @apollo/client의 useMutation 훅을 사용할 것.
            - 타입: @/commons/graphql/graphql에서 codegen으로 생성된 타입을 사용할 것.
            - 상태 관리: React의 useState 훅을 사용할 것.
            - 라우팅: next/navigation의 useRouter 훅을 사용할 것.
            - 폼: react-hook-form을 사용할 것 (선택사항)
            - 검증: zod를 사용할 것 (선택사항)

        2) 기능 요구사항
            - 1. (GQL) loginUser 뮤테이션을 정의할 것.
                - 파일 위치: src/components/auth/login/queries.ts 또는 적절한 위치
                - Mutation: loginUser
                - Variables: 
                    - email (String!)
                    - password (String!)
                - 응답데이터: 
                    - accessToken (JWT 토큰)
                    - [추가 응답 필드가 있다면 여기에 명시]
                - gql template literal 사용 (import { gql } from "@apollo/client")

            - 2. 로그인 기능 구현 (hook.ts)
                - "use client" 지시어 추가
                - useState를 사용하여 email, password 상태 관리
                - 로딩 상태 관리 (useState 또는 mutation의 loading)
                - 에러 상태 관리 (useState)
                - useMutation을 사용하여 loginUser 뮤테이션 실행
                - codegen으로 생성된 LoginUserDocument 타입 사용 (또는 직접 정의)
                - onChange 핸들러 함수들 생성
                    - onChangeEmail
                    - onChangePassword
                - onSubmit 함수 생성
                    - 입력값 검증 (이메일 형식, 비밀번호 필수)
                    - 뮤테이션 실행
                    - 성공 시 accessToken 저장
                        - 저장 위치: localStorage 또는 sessionStorage
                        - 키 이름: "accessToken" 또는 [원하는 키 이름]
                    - 로그인 성공 후 페이지 이동
                        - 이동 경로: "/" 또는 "/boards" 또는 [원하는 경로]
                    - 실패 시 에러 메시지 표시
                - 회원가입 페이지로 이동 함수 생성 (선택사항)
                    - 이동 경로: "/auth/signup" 또는 [원하는 경로]
                - 모든 상태와 핸들러 함수 반환

        3) 에러 처리 요구사항
            - 입력값 검증
                - 이메일 형식 검증 (정규식 또는 zod 사용)
                - 빈 값 체크
            - 로그인 실패 시 에러 메시지 표시
                - 에러 메시지 위치: UI 컴포넌트에 표시
            - 네트워크 에러 처리
            - 토큰 관련 에러 처리
