아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @01-common.mdc
            - @02-wireframe.mdc

==============================================

조건-파일경로) 구현될 TSX 파일경로: src/components/auth/signup/index.tsx
조건-파일경로) 구현될 CSS 파일경로: src/components/auth/signup/styles.module.css

==============================================

핵심요구사항) HTML과 flexbox를 활용한 와이어프레임 구조만 만들어낼 것.

핵심요구사항) 회원가입 폼의 기본 레이아웃 구조를 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.

        1) 레이아웃 구조
            - 1. wrapper 영역
                - 전체 회원가입 폼을 감싸는 컨테이너
                - 중앙 정렬

            - 2. 헤더 영역
                - 페이지 제목 ("회원가입")
                - 로고 이미지 또는 텍스트 (선택사항)

            - 3. 입력 영역
                - 이메일 입력 필드
                - 비밀번호 입력 필드
                - 비밀번호 확인 입력 필드
                - 이름 입력 필드
                - 전화번호 입력 필드 (선택사항)

            - 4. 약관 동의 영역
                - 전체 동의 체크박스
                - 개별 약관 동의 체크박스들
                  * 이용약관 동의 (필수)
                  * 개인정보 처리방침 동의 (필수)
                  * 마케팅 정보 수신 동의 (선택)

            - 5. 버튼 영역
                - 회원가입 버튼
                - 로그인 페이지로 이동 링크

        2) 스타일링 요구사항
            - CSS Modules 사용
            - flexbox 레이아웃만 사용 (position: absolute 금지)
            - 최소한의 스타일만 적용 (구조 위주)
            - 입력 필드 간격 일정하게 유지
            - 반응형 고려 (모바일/태블릿/데스크톱)

        3) 접근성 고려사항
            - 모든 input에 label 연결
            - placeholder 텍스트 제공
            - 필수 입력 필드 표시
