핵심요구사항) 회원가입 페이지 UI를 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.

        1) 핵심 UI 구조

            A. 헤더 영역
               - 페이지 제목: "회원가입"
               - 부제목: "My Trip Talk에 오신 것을 환영합니다"

            B. 폼 영역 (antd Form 사용)
               - 이메일 입력
                 * placeholder: "이메일을 입력하세요"
                 * validation: 이메일 형식 검증
                 * required: true

               - 비밀번호 입력
                 * placeholder: "비밀번호를 입력하세요"
                 * type: password
                 * validation: 최소 8자, 영문+숫자 조합
                 * required: true

               - 비밀번호 확인
                 * placeholder: "비밀번호를 다시 입력하세요"
                 * type: password
                 * validation: 비밀번호 일치 확인
                 * required: true

               - 이름 입력
                 * placeholder: "이름을 입력하세요"
                 * required: true

               - 전화번호 입력 (선택)
                 * placeholder: "010-1234-5678"
                 * required: false

            C. 약관 동의 영역
               - 전체 동의 체크박스
               - 개별 약관 체크박스들:
                 * [필수] 이용약관 동의
                 * [필수] 개인정보 처리방침 동의
                 * [선택] 마케팅 정보 수신 동의

            D. 버튼 영역
               - 회원가입 버튼 (primary, block)
               - 로그인 페이지 이동 링크

        2) 스타일링 요구사항
            - CSS Modules 사용 (styles.module.css)
            - 폼 최대 너비: 400px
            - 중앙 정렬: margin: 0 auto
            - 입력 필드 간격: 16px
            - 버튼 높이: 48px
            - 카드 스타일 적용 (배경: #fff, 그림자)
            - 반응형 디자인: 모바일에서 padding 조정

        3) Hook 연동 (hook.ts)
            - useSignup hook 구현
            - 상태 관리:
              * email: string
              * password: string
              * passwordConfirm: string
              * name: string
              * phone: string
              * agreedToTerms: boolean
              * agreedToPrivacy: boolean
              * agreedToMarketing: boolean
            - 함수:
              * handleSignup: 회원가입 실행
              * handleAllAgree: 전체 동의 토글
              * validatePassword: 비밀번호 유효성 검증
              * validateEmail: 이메일 유효성 검증

        4) 에러 처리
            - 입력 validation 에러 표시
            - API 호출 실패 시 에러 메시지
            - 중복 이메일 체크
