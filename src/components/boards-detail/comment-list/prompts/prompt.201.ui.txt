핵심요구사항) 댓글 목록 UI 스타일을 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것
            (wireframe에서 정의한 구조를 유지하며, 시각적 스타일만 적용)

        1) 색상 (Colors)

            - 배경색
              * 댓글 아이템 배경: #ffffff
              * 구분선 배경: #e4e4e4

            - 텍스트 색상
              * 작성자 이름: #000000
              * 댓글 내용: #333333
              * 작성일시: #999999

            - 별점 색상
              * 활성 별: #fadb14 (Ant Design 기본)

        2) 크기 (Sizes)

            - 프로필 이미지
              * width: 40px
              * height: 40px
              * border-radius: 50% (원형)

            - 아이콘 크기
              * 수정 아이콘: 20px x 20px
              * 삭제 아이콘: 20px x 20px

            - 폰트 크기
              * 작성자 이름: 16px
              * 댓글 내용: 14px
              * 작성일시: 12px

            - 별점 크기
              * Ant Design Rate size: default (20px)

        3) 간격 (Spacing)

            - padding
              * 댓글 아이템: 16px 0

            - margin
              * 작성자 이름 우측: 12px
              * 댓글 내용 상단: 8px
              * 작성일시 상단: 8px

            - gap
              * 상단 영역 요소 간격: 12px
              * 액션 버튼 간격: 8px

        4) 테두리 (Borders)

            - 구분선
              * width: 100%
              * height: 1px
              * background: #e4e4e4
              * margin: 16px 0

        5) 타이포그래피 (Typography)

            - font-weight
              * 작성자 이름: 600
              * 댓글 내용: 400
              * 작성일시: 400

            - line-height
              * 댓글 내용: 1.6

        6) 상태별 스타일 (State Styles)

            - 아이콘 버튼
              * 기본: opacity: 0.7
              * hover: opacity: 1, cursor: pointer
              * transition: opacity 0.2s

            - 댓글 아이템
              * hover: background: #f9f9f9

        7) 이미지 및 아이콘 경로
            - 프로필 이미지: @/assets/profile_image.png
            - 수정 아이콘: @/assets/pencil.png
            - 삭제 아이콘: @/assets/trashbin.png
            - Next.js Image 컴포넌트 사용
              * width, height 명시
              * alt 속성 추가

        8) 데이터 바인딩
            - hook.ts의 useCommentList 훅 사용
            - fetchBoardComments 데이터 map으로 순회
            - 각 댓글: writer, rating, contents, createdAt 표시
            - 작성일시 포맷: YYYY.MM.DD

        9) 기능 연동
            - 수정 아이콘 클릭 이벤트 (향후 구현)
            - 삭제 아이콘 클릭 이벤트 (향후 구현)

        10) 스타일링 요구사항 (상세)
            - CSS Modules 사용 (styles.module.css)
            - wireframe에서 정의한 레이아웃 구조 유지
            - 구조 변경 금지 (flex, gap 등)

        11) 주의사항
            ⚠️ UI 단계에서 변경하지 말 것:
            - HTML 구조
            - 레이아웃 방식 (flex)
            - 요소 배치 방향 (row, column)
            - 정렬 방식 (align-items)

            ✅ UI 단계에서 반드시 포함할 것:
            - 모든 색상 값
            - 모든 크기 값
            - 모든 간격 값 (padding, margin, gap 수치)
            - 테두리 스타일
            - 타이포그래피
            - 상태별 스타일 (hover)
