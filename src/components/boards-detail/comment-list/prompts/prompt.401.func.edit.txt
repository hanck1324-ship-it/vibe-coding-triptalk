조건-파일경로) (참고) 공통 GQL 타입: src/commons/graphql/graphql.ts (codegen으로 생성된 타입 사용)

==============================================

핵심요구사항) 댓글 수정 기능을 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.

        1) 라이브러리조건
            - API: @apollo/client의 useMutation 훅을 사용할 것.
            - 타입: @/commons/graphql/graphql에서 codegen으로 생성된 타입을 사용할 것.
            - 상태 관리: React의 useState 훅을 사용할 것.

        2) 기능 요구사항
            - 1. (GQL) src/components/boards-detail/comment-list/queries.ts 파일에 updateBoardComment 뮤테이션을 정의할 것.
                - Mutation: updateBoardComment
                - Variables: commentId (ID!), password (String!), updateBoardCommentInput (contents, rating)
                - 응답데이터: _id, contents, rating, updatedAt
                - gql template literal 사용 (import { gql } from "@apollo/client")

            - 2. 댓글 수정 UI 모드
                - 수정 아이콘 클릭 시 해당 댓글을 수정 모드로 전환
                - 수정 모드에서 textarea와 별점 컴포넌트 표시
                - 비밀번호 입력 필드 추가
                - 저장/취소 버튼 표시

            - 3. (HOOK) 댓글 수정 기능 구현
                - "use client" 지시어 추가
                - 수정 중인 댓글 ID 상태 관리
                - 수정 폼 상태 관리 (contents, rating, password)
                - updateBoardComment 뮤테이션 실행
                - 패스워드 검증
                - 성공 시 댓글 목록 refetch
                - 수정 모드 취소 기능

        3) 에러 처리 요구사항
            - 패스워드 불일치 시 에러 처리
            - 뮤테이션 실패 시 에러 메시지 표시
            - 유효성 검사 (내용 비어있지 않음)

        4) 최적화 요구사항
            - refetchQueries를 사용하여 댓글 목록 자동 갱신
            - 불필요한 리렌더링 방지
