핵심요구사항) 게시글 상세 조회 기능을 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.

        1) 라이브러리조건
            - API: @apollo/client의 useQuery 훅을 사용할 것.
            - 타입: @/commons/graphql/graphql에서 codegen으로 생성된 타입을 사용할 것.
            - 라우팅: next/navigation의 useRouter, useParams 훅을 사용할 것.

        2) 기능 요구사항
            - 1. (GQL) src/components/boards-detail/detail/queries.ts 파일에 fetchBoard 쿼리를 정의할 것.
                - Query: fetchBoard
                - Variables: boardId (ID!)
                - 응답데이터: _id, writer, title, contents, youtubeUrl, likeCount, dislikeCount, images, boardAddress, user, createdAt, updatedAt, deletedAt
                - gql template literal 사용 (import { gql } from "@apollo/client")

            - 2. (HOOK) src/components/boards-detail/detail/hook.ts 파일에 커스텀 훅을 만들 것.
                - "use client" 지시어 추가
                - useQuery를 사용하여 fetchBoard 쿼리 요청
                - codegen으로 생성된 FetchBoardDocument 타입 사용
                - useParams를 사용하여 boardId 가져오기
                - boardId가 없을 경우 쿼리 실행을 skip하도록 처리
                - loading 상태 반환
                - board 데이터 반환
                - 수정하기 페이지로 이동하는 함수 반환 (useRouter의 push 메서드 사용)

        3) 에러 처리 요구사항
            - 로딩 중 상태 표시
            - 데이터가 없을 경우 처리
            - 에러 발생 시 처리

        4) 최적화 요구사항
            - 동적 import 사용 (react-player는 ssr: false 옵션)
            - 불필요한 리렌더링 방지
