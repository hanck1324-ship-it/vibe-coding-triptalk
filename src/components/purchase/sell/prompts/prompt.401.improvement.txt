핵심요구사항) 이미지 업로드 미리보기 기능을 step-by-step 으로 개선할 것.

        1) 이미지 미리보기 기능 개선
            - 이미지 선택 시 즉시 미리보기 표시
              * 사용자가 파일을 선택하면 선택한 이미지를 즉시 확인할 수 있도록 개선
              * 임시 URL(Blob URL)을 생성하여 미리보기 제공
              * URL.createObjectURL()을 사용하여 File 객체로부터 임시 URL 생성
              * 생성된 임시 URL을 상태에 저장하여 UI에 표시
            
            - 미리보기 URL 관리
              * 미리보기 URL 배열 상태로 관리
              * 이미지 추가 시: File 객체와 함께 임시 URL 생성 및 배열에 추가
              * 이미지 삭제 시: 해당 임시 URL 메모리 해제 후 배열에서 제거
              * 컴포넌트 언마운트 시: 모든 생성된 임시 URL 메모리 해제 처리
            
            - 메모리 관리
              * URL.revokeObjectURL()을 사용하여 사용하지 않는 임시 URL 해제
              * 이미지 삭제 시 해당 URL 즉시 해제
              * 컴포넌트 언마운트 시 모든 URL 정리 (useEffect cleanup)
            
            - UI 연동
              * 미리보기 URL 배열을 컴포넌트에 전달
              * 각 미리보기 URL을 Image 컴포넌트로 렌더링
              * 이미지 선택 직후 즉시 미리보기 표시되도록 처리

        2) 구현 시 주의사항
            - 임시 URL 생성 타이밍
              * 파일 선택 직후 즉시 URL 생성
              * 파일 검증(타입, 크기) 통과 후 URL 생성
              * URL 생성 실패 시 에러 처리
            
            - 메모리 누수 방지
              * 삭제된 이미지의 URL은 반드시 해제
              * 컴포넌트 언마운트 시 모든 URL 정리
              * 동일한 파일 재선택 시에도 이전 URL 해제 후 새로 생성
            
            - 상태 동기화
              * File 객체 배열과 미리보기 URL 배열의 인덱스 일치 보장
              * 이미지 삭제 시 두 배열 모두 동일하게 업데이트
              * 배열 순서 유지
