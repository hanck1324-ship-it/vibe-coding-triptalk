핵심요구사항) 숙박권 상세 페이지 기능을 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.

        1) GraphQL 쿼리 구현
            - fetchTravelproduct 쿼리를 사용하여 서버에서 여행 상품 상세 데이터 조회
            - 상품 ID(travelproductId)를 쿼리 변수로 전달하여 해당 상품의 상세 정보 조회
            - fetchUserLoggedIn 쿼리를 사용하여 현재 로그인한 사용자의 포인트 정보 조회
            - mock 데이터 대신 실제 데이터베이스의 최신 정보 사용
            - 로딩 상태와 에러 상태 관리하여 사용자에게 적절한 피드백 제공

        2) 상태 관리 구현
            - URL 파라미터에서 숙박권 ID 추출
            - 구매 모달 열림/닫힘 상태 관리
            - 포인트 부족 모달 열림/닫힘 상태 관리
            - 사용자 보유 포인트 정보 관리
            - 에러 메시지 상태 관리
            - 필요 포인트 계산 (상품 가격 기반)

        3) 이벤트 핸들러 구현
            - 구매 모달 열기 핸들러
            - 구매 모달 닫기 핸들러
            - 포인트 부족 모달 닫기 핸들러
            - 구매 확인 핸들러 (핵심 로직)
                - 포인트 체크: 사용자 보유 포인트와 필요 포인트 비교
                - 포인트 부족 시: 포인트 부족 모달 표시, 구매 프로세스 중단
                - 포인트 충분 시: 구매 API 호출
                - 구매 성공 시: 구매 모달 닫기, 판매 등록 화면으로 이동
                - 구매 실패 시: 에러 메시지 표시, 구매 모달 닫기

        4) 구매 확인 로직 흐름
            - 구매 확인 버튼 클릭 시 포인트 체크
            - 포인트 부족 시: 포인트 부족 모달 표시, 구매 프로세스 중단
            - 포인트 충분 시: 구매 API 호출
            - 구매 성공 시: 구매 모달 닫기, 판매 등록 화면으로 이동
            - 구매 실패 시: 에러 메시지 표시, 구매 모달 닫기

        5) 주의사항
            - 포인트 부족 시 구매 프로세스를 즉시 중단해야 함
            - 구매 API 호출은 비동기로 처리하고 에러 처리가 필요함
            - GraphQL 쿼리로 조회한 데이터는 로딩 상태를 고려하여 사용해야 함
