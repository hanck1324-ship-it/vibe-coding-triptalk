핵심요구사항) 숙박권 상세 페이지에서 빠진 상세 정보들을 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.

        1) 라이브러리조건
            - 아이콘: @ant-design/icons (HeartOutlined, HeartFilled, EnvironmentOutlined, UserOutlined)
            - 지도: 다음 지도 API 또는 Kakao 지도 API
            - 이미지: Next.js Image 컴포넌트

        2) 숙박권 상세 정보 확장

            [A] 이미지 갤러리
                - 여러 이미지 슬라이더 구현
                    * travelproduct.images 배열 활용
                    * 메인 이미지: 큰 이미지 (full width, 500px height)
                    * 썸네일 이미지: 하단에 작은 이미지들 (5개까지 표시)
                    * 썸네일 클릭 시 메인 이미지 변경
                
                - 좌우 화살표 버튼
                    * 이전/다음 이미지로 이동
                    * 첫 이미지/마지막 이미지에서는 비활성화
                
                - 이미지 카운터
                    * 현재 이미지 / 전체 이미지 수 표시
                    * 예: "1 / 5"

            [B] 숙박권 기본 정보
                - 제목 (name)
                    * font-size: 32px
                    * font-weight: 700
                    * color: #1a1a1a
                
                - 가격 (price)
                    * font-size: 28px
                    * font-weight: 700
                    * color: #2974e5
                
                - 태그 (tags)
                    * 태그 배지들 표시
                    * 예: "1인 전용", "아파트", "반신욕&스파"
                    * 스타일: padding: 6px 12px, background: #f0f0f0, border-radius: 16px
                
                - 북마크 버튼
                    * 우측 상단에 배치
                    * HeartOutlined / HeartFilled 아이콘
                    * 클릭 시 북마크 토글
                    * localStorage에 저장

            [C] 숙소 상세 설명
                - 설명 (remarks)
                    * 짧은 설명 (한 줄)
                    * font-size: 18px
                    * color: #666
                
                - 상세 내용 (contents)
                    * 긴 설명 (여러 줄)
                    * font-size: 16px
                    * line-height: 1.8
                    * white-space: pre-wrap (줄바꿈 유지)
                
                - 생성일/수정일
                    * createdAt, updatedAt 표시
                    * 예: "등록일: 2024.01.15"
                    * 예: "수정일: 2024.01.20"

            [D] 판매자 정보
                - 판매자 프로필
                    * 프로필 이미지 (seller.picture)
                    * 판매자 이름 (seller.name)
                    * 판매자 이메일 (seller.email)
                    * 레이아웃: 가로 배치 (flexbox)
                
                - 판매자 카드 스타일
                    * padding: 20px
                    * border: 1px solid #e8e8e8
                    * border-radius: 12px
                    * background: #fafafa

            [E] 숙소 주소 정보
                - 주소 표시
                    * travelproductAddress.address (기본 주소)
                    * travelproductAddress.addressDetail (상세 주소)
                    * travelproductAddress.zipcode (우편번호)
                    * EnvironmentOutlined 아이콘 추가
                
                - 지도 표시
                    * travelproductAddress.lat (위도)
                    * travelproductAddress.lng (경도)
                    * 카카오 지도 또는 다음 지도 API 사용
                    * 지도 높이: 400px
                    * 마커 표시

            [F] 판매 상태 표시
                - 판매 완료 여부
                    * soldAt이 있으면 "판매 완료"
                    * soldAt이 없으면 "판매 중"
                    * 판매 완료 시 구매 버튼 비활성화
                
                - 구매자 정보 (판매 완료 시)
                    * buyer.name
                    * buyer.email
                    * "이 상품은 이미 판매되었습니다" 메시지 표시

        3) 상호작용 기능

            [A] 북마크 기능
                - Hook 로직 (hook.ts)
                    * isBookmarked 상태 관리 (boolean)
                    * 초기값: localStorage에서 'bookmarkedProducts' 확인
                    * handleToggleBookmark 함수
                    * localStorage 업데이트
                
                - UI 연동 (index.tsx)
                    * 북마크 버튼 추가 (우측 상단)
                    * isBookmarked ? <HeartFilled /> : <HeartOutlined />
                    * onClick={handleToggleBookmark}

            [B] 이미지 갤러리
                - Hook 로직 (hook.ts)
                    * currentImageIndex 상태 관리 (number, 초기값: 0)
                    * handleNextImage 함수: index + 1 (최대값 제한)
                    * handlePrevImage 함수: index - 1 (최소값 0)
                    * handleSelectImage 함수: 특정 인덱스로 이동
                
                - UI 연동 (index.tsx)
                    * 좌우 화살표 버튼 추가
                    * 썸네일 이미지 클릭 핸들러
                    * 이미지 카운터 표시

            [C] 공유 기능
                - 공유 버튼 추가
                    * URL 복사 기능
                    * navigator.clipboard.writeText 사용
                    * 복사 완료 메시지 표시 (Toast 또는 Alert)

        4) 스타일링 요구사항

            [A] 레이아웃 구조
                - 좌우 2단 레이아웃 (Desktop)
                    * 좌측: 이미지 갤러리 (60%)
                    * 우측: 상세 정보 (40%)
                
                - 1단 레이아웃 (Mobile)
                    * 이미지 갤러리
                    * 상세 정보 (순서대로 세로 배치)

            [B] 섹션별 스타일
                - .detailContainer
                    * max-width: 1200px
                    * margin: 0 auto
                    * padding: 40px 20px
                
                - .imageGallery
                    * flex: 1
                    * max-width: 700px
                
                - .infoSection
                    * flex: 1
                    * max-width: 500px
                    * padding: 0 40px
                
                - .sellerCard
                    * padding: 20px
                    * border: 1px solid #e8e8e8
                    * border-radius: 12px
                    * background: #fafafa
                
                - .addressSection
                    * padding: 24px 0
                    * border-top: 1px solid #e8e8e8
                
                - .mapContainer
                    * width: 100%
                    * height: 400px
                    * border-radius: 12px
                    * overflow: hidden

            [C] 반응형 디자인
                - Desktop (기본)
                    * 좌우 2단 레이아웃
                    * max-width: 1200px
                
                - Tablet (@media max-width: 1024px)
                    * 좌우 비율 조정: 50% / 50%
                    * 패딩 축소
                
                - Mobile (@media max-width: 768px)
                    * 1단 레이아웃 (세로 배치)
                    * 이미지 갤러리: 100%
                    * 정보 섹션: 100%

        5) GraphQL 쿼리 확장

            [A] queries.ts 업데이트
                - FETCH_TRAVELPRODUCT 쿼리에 필드 추가
                    * contents (상세 설명)
                    * createdAt (생성일)
                    * updatedAt (수정일)
                    * pickedCount (찜 개수)
                    * travelproductAddress { address, addressDetail, zipcode, lat, lng }
                    * seller { _id, name, email, picture }
                    * buyer { _id, name, email }

        6) 사용자 경험 개선

            [A] 로딩 상태
                - 이미지 로딩 중: Skeleton UI 표시
                - 데이터 로딩 중: "불러오는 중..." 메시지
                
            [B] 에러 처리
                - 이미지 로드 실패: fallback 이미지 표시
                - 데이터 조회 실패: "상품을 찾을 수 없습니다" 메시지
                - 지도 로드 실패: "지도를 불러올 수 없습니다" 메시지
                
            [C] 접근성
                - 이미지 alt 속성 추가
                - 버튼 aria-label 추가
                - 키보드 네비게이션 지원 (화살표 키로 이미지 이동)
