핵심요구사항) 숙박권 리뷰 작성 컴포넌트 UI를 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.

        1) 핵심 UI 구조

            A. 작성자 정보 영역
               - 프로필 이미지 (작은 원형)
               - 사용자 이름

            B. 평점 입력 영역
               - 별점 (antd Rate 컴포넌트)
               - 5점 만점
               - 기본값: 5점

            C. 리뷰 내용 입력 영역
               - Textarea (antd Input.TextArea)
               - placeholder: "숙소에 대한 솔직한 후기를 남겨주세요"
               - 최소 높이: 120px
               - 자동 리사이즈
               - 글자 수 표시 (0/500자)

            D. 버튼 영역
               - 등록 버튼 (primary)
               - 취소 버튼 (default)

        2) 스타일링 요구사항
            - CSS Modules 사용 (styles.module.css)
            - 카드 스타일 적용 (배경: #fff, border, padding)
            - 프로필 이미지: 32x32px, border-radius: 50%
            - 버튼 간격: 8px
            - 전체 padding: 20px
            - 반응형: 모바일에서 버튼 세로 배치

        3) Hook 연동 (hook.ts)
            - useCommentWrite hook 구현
            - Props:
              * accommodationId: number (숙박권 ID)
              * onSuccess?: () => void (등록 성공 콜백)
            - 상태 관리:
              * rating: number (1-5)
              * content: string
              * isSubmitting: boolean
            - 함수:
              * handleSubmit: 리뷰 등록 실행
              * handleCancel: 입력 내용 초기화
              * setRating: 평점 변경
              * setContent: 내용 변경

        4) Validation
            - 평점: 1점 이상 필수
            - 내용: 10자 이상 필수
            - 내용: 500자 이하
            - Validation 실패 시 에러 메시지 표시

        5) 에러 처리
            - API 호출 실패 시 에러 메시지
            - 네트워크 에러 처리
            - 권한 에러 처리 (로그인 필요)
