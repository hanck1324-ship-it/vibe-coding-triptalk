핵심요구사항) 포인트 부족 알림 모달 UI를 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.

        1) UI 구조 요구사항
            
            [A] 모달 기본 구조
                - Ant Design Modal 컴포넌트 사용
                - Props: isOpen, onClose, currentPoint, requiredPoint
                - title: "포인트 부족"
            
            [B] 알림 메시지
                - "포인트가 부족합니다." 메시지 표시
                - 에러/경고 색상으로 강조
                - 중앙 정렬
            
            [C] 포인트 정보 표시
                - 보유 포인트 (currentPoint)
                - 필요 포인트 (requiredPoint)
                - 부족 포인트 (계산: requiredPoint - currentPoint)
                - 각 항목을 라벨-값 쌍으로 표시
                - 부족 포인트는 강조 (빨간색)
            
            [D] 버튼 영역
                - 닫기 버튼: onClose() 호출
                - 충전하러 가기 버튼:
                  - 마이페이지 포인트 탭으로 라우팅
                  - 경로: /myPage?tab=point
                  - useRouter의 push 메서드 사용
                  - 라우팅 후 모달 닫기

        2) 데이터 표시 요구사항
            - 포인트 값은 천 단위 콤마 구분 (toLocaleString)
            - "P" 단위 표시
            - 부족 포인트는 계산하여 표시

        3) 스타일링 요구사항
            - CSS Modules 사용
            - 모달 너비: 400px (작은 알림 모달)
            - 포인트 정보 영역:
              - 배경색 (연한 회색)
              - 패딩 및 border-radius
              - flex 레이아웃 (좌우 정렬)
            - 부족 포인트 강조:
              - 상단 border로 구분
              - 빨간색 텍스트
              - 볼드체

        4) Props 인터페이스
            - isOpen: boolean (모달 표시 여부)
            - onClose: () => void (닫기 콜백)
            - currentPoint: number (현재 보유 포인트)
            - requiredPoint: number (필요한 포인트)

        5) 테스트 체크리스트
            - [ ] 포인트 부족 시 모달이 표시되는가?
            - [ ] 보유/필요/부족 포인트가 정확하게 표시되는가?
            - [ ] 포인트 값에 천 단위 콤마가 표시되는가?
            - [ ] 충전하러 가기 버튼 클릭 시 마이페이지로 이동하는가?
            - [ ] 닫기 버튼으로 모달이 닫히는가?
