핵심요구사항) 다음의 조건에 맞게 게시글 목록 기능을 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.

        1) 라이브러리조건
            - API: @apollo/client의 useQuery 훅을 사용할 것.
            - 타입: @/commons/graphql/graphql에서 codegen으로 생성된 타입을 사용할 것.
            - 라우팅: next/navigation의 useRouter, useSearchParams 훅을 사용할 것.
            - 상태 관리: React의 useState 훅을 사용할 것.

        2) 기능 요구사항

            - 1. (GQL) src/components/boards-list/quires.ts 파일에 쿼리와 뮤테이션을 정의할 것.
                
                A. fetchBoards 쿼리:
                - Query: fetchBoards
                - Variables: page (Int), search (String, 선택사항)
                - 응답데이터: _id, writer, title, createdAt
                - gql template literal 사용 (import { gql } from "@apollo/client")
                
                B. deleteBoard 뮤테이션:
                - Mutation: deleteBoard
                - Variables: boardId (ID!), password (String!)
                - 응답데이터: _id (삭제된 게시글 ID)
                - gql template literal 사용

            - 2. (HOOK) src/components/boards-list/hook.ts 파일에 커스텀 훅을 만들 것.
                - "use client" 지시어 추가
                - useSearchParams로 URL 쿼리 파라미터 읽기
                - 'page' 값을 관리하는 state (useState, 기본값: URL 쿼리 또는 1)
                - 'search' 값을 관리하는 state (useState, 기본값: "")
                - 'selectedCategory' 값을 관리하는 state (useState, 기본값: "all")
                
                - useQuery를 사용하여 fetchBoards 쿼리 요청
                - codegen으로 생성된 FetchBoardsDocument 타입 사용
                - loading, error 상태 처리
                
                - 페이지네이션 함수:
                    * onClickPrevPage: page > 1일 때만 작동, URL 쿼리 업데이트
                    * onClickNextPage: URL 쿼리 업데이트
                    * onClickPage: 특정 페이지 번호로 이동
                
                - 검색 함수:
                    * onChangeSearchInput: 검색어 입력 처리
                    * onClickSearch: 검색 실행, page를 1로 리셋
                    * onPressEnter: Enter 키 입력 시 검색 실행
                
                - 카테고리 필터 함수:
                    * onClickCategory: 선택된 카테고리 변경, page를 1로 리셋
                
                - 라우팅 함수:
                    * onClickBoard: 게시글 ID를 받아 /boards/[boardId]로 이동
                    * onClickWrite: 게시글 작성 페이지로 이동 (/boards/new)
                
                - 삭제 함수:
                    * hoveredBoardId: string | null (hover 중인 게시글 ID 상태)
                    * isDeleteModalOpen: boolean (삭제 모달 열림 상태)
                    * selectedBoardId: string | null (삭제할 게시글 ID)
                    * deletePassword: string (삭제 비밀번호 입력값)
                    
                    * onMouseEnter: (boardId) => hover 시 hoveredBoardId 설정
                    * onMouseLeave: () => hoveredBoardId를 null로 설정
                    
                    * onClickDelete: (boardId) => 삭제 아이콘 클릭 시
                      - 이벤트 버블링 중단 (e.stopPropagation())
                      - selectedBoardId 설정
                      - 삭제 모달 열기
                    
                    * onChangeDeletePassword: 비밀번호 입력 핸들러
                    
                    * onConfirmDelete: 삭제 확인 시
                      - useMutation으로 deleteBoard 실행
                      - Variables: { boardId: selectedBoardId, password: deletePassword }
                      - 성공 시:
                        * 목록 refetch
                        * 모달 닫기
                        * 비밀번호 초기화
                        * 성공 메시지 표시 (선택사항)
                      - 실패 시:
                        * 에러 메시지 표시 ("비밀번호가 일치하지 않습니다." 등)
                    
                    * onCancelDelete: 삭제 취소 시
                      - 모달 닫기
                      - 상태 초기화
                
                - URL 쿼리 파라미터 동기화:
                    * useRouter의 push 메서드로 URL 업데이트
                    * 쿼리 문자열: ?page=2&search=제주도&category=review

            - 3. (UI) src/components/boards-list/index.tsx 파일에 UI를 구현할 것.
                - "use client" 지시어 추가
                - hook.ts에서 만든 훅을 사용하여 데이터를 바인딩할 것.
                - fetchBoards로 받아온 데이터를 목록 형태로 렌더링할 것.
                - 각 행(item)에 data-testid="board-item-[index]"를 부여할 것.
                - 각 행(item) 클릭 시 훅에서 만든 라우팅 함수를 실행할 것.
                
                - 삭제 아이콘 UI (Figma 노드 394:11698 기준):
                    * import: @ant-design/icons의 DeleteOutlined 아이콘 사용
                    * 위치: 각 테이블 행의 우측 끝 (날짜 컬럼 뒤)
                    * 크기: 24x24px
                    * 색상: #333333 (기본), #ff4d4f (hover)
                    * 표시 조건: hoveredBoardId === board._id일 때만 표시
                    * data-testid="delete-button-[index]"
                    
                    * 이벤트:
                      - onClick={(e) => { e.stopPropagation(); onClickDelete(board._id); }}
                      - onMouseEnter={() => onMouseEnter(board._id)}
                      - onMouseLeave={onMouseLeave}
                    
                    * 스타일:
                      - cursor: pointer
                      - transition: opacity 0.2s
                      - opacity: 0 (기본), opacity: 1 (hover)
                    
                - 삭제 확인 모달 (Ant Design Modal):
                    * title: "게시글 삭제"
                    * open: isDeleteModalOpen
                    * onOk: onConfirmDelete
                    * onCancel: onCancelDelete
                    * okText: "삭제"
                    * cancelText: "취소"
                    * okButtonProps: {{ danger: true }}
                    
                    * 내용:
                      - 안내 메시지: "게시글을 삭제하시겠습니까?"
                      - 비밀번호 입력 필드:
                        * type="password"
                        * placeholder="비밀번호를 입력하세요"
                        * value={deletePassword}
                        * onChange={onChangeDeletePassword}
                
                - 검색 UI:
                    * input에 data-testid="search-input"
                    * button에 data-testid="search-button"
                    * onChange, onClick, onKeyPress 이벤트 연결
                
                - 카테고리 필터 UI:
                    * 각 버튼에 data-testid="category-filter-[categoryId]"
                    * 선택된 카테고리 활성화 스타일 적용
                
                - 페이지네이션 UI:
                    * 이전 버튼: data-testid="pagination-prev-button"
                    * 다음 버튼: data-testid="pagination-next-button"
                    * 페이지 번호 버튼: data-testid="pagination-page-[number]"
                    * 현재 페이지 강조 표시
                    * 총 페이지 수 계산 (전체 게시글 수 / 10)
                
                - 등록 버튼:
                    * data-testid="write-button"
                    * 클릭 시 /boards/new로 이동
                
                - CSS Modules 사용 (styles.module.css)

            - 4. (페이지) src/app/boards/page.tsx 파일을 업데이트할 것.
                - BoardsList 컴포넌트 import
                - 간단한 레이아웃으로 감싸기

        3) 에러 처리 요구사항
