핵심요구사항) 포인트 충전 페이지의 와이어프레임 구조를 구현할 것
            (구조와 배치만 정의, 색상/크기/폰트 등 스타일은 prompt.201에서 처리)

        1) HTML 구조 및 레이아웃 배치

            A. 페이지 헤더 섹션
               - 페이지 제목: "포인트 충전"
               - 구분선

            B. 현재 포인트 표시 섹션
               - 라벨: "현재 보유 포인트"
               - 포인트 값 표시 (예: 10,000 P)
               - 구분선

            C. 충전 금액 선택 섹션 ⭐핵심⭐
               - 라벨: "충전 금액 선택" + 필수 표시(*)
               - 금액 버튼 그리드
                 * 레이아웃: display: grid
                 * 데스크톱: 4열 (grid-template-columns: repeat(4, 1fr))
                 * 태블릿(768px~1023px): 2열 (repeat(2, 1fr))
                 * 모바일(767px 이하): 2열 (repeat(2, 1fr))
                 * gap 적용
                 * 옵션:
                   - 100원
                   - 500원
                   - 2,000원
                   - 5,000원
                   - 10,000원
                   - 50,000원
               - 구분선

            D. 버튼 영역
               - 레이아웃: display: flex
               - 정렬: 오른쪽 정렬 (justify-content: flex-end)
               - 버튼 배치: 가로 나열 (flex-direction: row)
               - 버튼 간격: gap 적용
               - 버튼 종류:
                 * 취소 버튼
                 * 충전하기 버튼 (선택 시 활성화)

        2) 기술 요구사항
            - CSS Modules 사용 (styles.module.css)
            - flexbox, grid 레이아웃 사용
            - 구조와 배치만 구현 (레이아웃 방식, flex/grid 설정)
            - 스타일링 요소는 제외 (색상, 크기, 폰트 등은 prompt.201 참조)

        3) 주의사항
            ⚠️ wireframe 단계에서 포함하지 말 것:
            - 색상 값 (hex, rgb 등)
            - 크기 값 (px, rem 등의 구체적 수치)
            - padding, margin 구체적 수치
            - font-size, font-weight
            - border-radius 값
            - 상태별 스타일 (hover, focus, disabled)

            ✅ wireframe 단계에서 반드시 포함할 것:
            - display 속성 (flex, grid, block 등)
            - flex/grid 레이아웃 설정
            - 요소 배치 방향 (row, column)
            - 정렬 방식 (justify-content, align-items)
            - 반응형 구조 변경 (열 수 변경 등)
