아래의 조건을 모두 적용하여, 아래의 요구사항을 모두 구현할 것.
구현 결과를 체크리스트로 반환할 것.

==============================================

조건-커서룰) 아래의 커서룰을 적용하여 작업하고, 이 작업이 끝나면 해당 rules 적용 결과를 체크리스트로 반환할 것.
            - @01-common.mdc
            - @04-func.mdc

==============================================

조건-파일경로) 구현될 TSX 파일경로: src/components/mypage/navigation/index.tsx
조건-파일경로) 구현될 CSS 파일경로: src/components/mypage/navigation/styles.module.css

==============================================

핵심요구사항) 마이페이지 네비게이션 기능을 step-by-step 으로 구현하고, 적용 결과를 체크리스트로 반환할 것.

        1) 타입 정의
            - MyPageTab 타입 정의: "profile" | "point" | "history"
            - INavigationProps 인터페이스 정의

        2) Props 인터페이스
            - activeTab: MyPageTab (필수) - 현재 활성화된 탭
            - onTabChange: (tab: MyPageTab) => void (필수) - 탭 변경 핸들러
            - userName?: string (선택, 기본값: "사용자") - 사용자 이름
            - userPoint?: number (선택, 기본값: 0) - 사용자 포인트

        3) 탭 상태 표시
            - activeTab prop을 받아서 현재 활성화된 탭 확인
            - activeTab과 일치하는 메뉴 항목에 active 클래스 적용

        4) 메뉴 항목 클릭 핸들러
            - 거래내역&북마크 버튼 클릭 시: onTabChange("history") 호출
            - 포인트 사용 내역 버튼 클릭 시: onTabChange("point") 호출
            - 비밀번호 변경 버튼 클릭 시: onTabChange("profile") 호출

        5) 활성 상태 표시
            - activeTab과 일치하는 메뉴 항목에 active 클래스 적용
            - active 상태일 때 텍스트 색상: #262626 (진한 색상)
            - active 상태일 때 font-weight: 600 (진하게)
            - active 상태일 때 배경색: #f5f5f5
            - active 상태일 때 화살표 아이콘 색상: #262626 (진한 색상)

        6) 데이터 표시
            - userName prop을 프로필 이름으로 표시
            - userPoint prop을 포인트 값으로 표시 (toLocaleString()을 사용하여 천 단위 구분 기호 적용)
